# 单例

保证一个类仅有一个实例，并提供一个访问它的全局访问点

场景：线程池、全局缓存、浏览器中的window对象...

实现方法：用一个变量来标记当前是否已经为某个类创建过对象，如果是，则在下一次获取改类的实例时，直接返回之前创建的对象

# 惰性单例

指在需要的时候才创建对象实例

class Person {
  constructor(name, age) {
    this.name = name
    this.age = age
  }
  say() {
    return `i am ${this.name}, ${this.age} years old`
  }
}

let getSingle = function(Func) {
  let instance = null
  return function(...args) {
    return instance || (instance = new Func(...args))
  }
}

let getPersonSingle = getSingle(Person)
console.log(getPersonSingle('coderly', 18).say()) // i am coderly, 18 years old
console.log(getPersonSingle('李白', 1320).say()) // i am coderly, 18 years old
console.log(getPersonSingle('杜甫', 1309).say()) // i am coderly, 18 years old

# 行为型设计模式

提示

用于描述 “类或对象之间怎样相互协作共同完成单个对象无法单独完成的任务”

# 策略

定义一系列的算法，把它们一个个封装起来，并且使它们可以相互替换

简单例子

const operations = {
  remove(row) {
    // do something
    return 'remove execute'
  },
  edit(row) {
    // do something
    return 'edit execute'
  },
  add(row) {
    // do something
    return 'add execute'
  }
}

function handleRowClick(row) {
  return row.type && typeof operations[row.type] === 'function' && operations[row.type](row)
}
